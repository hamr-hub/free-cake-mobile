# Free Cake Mobile - C端移动应用需求

## 一、项目背景
Free Cake 村镇 AI 蛋糕项目已完成 Web B端后台和 Rust Axum 后端开发，现需开发 C端移动应用（iOS + Android），让村镇用户可以通过手机端参与 AI 蛋糕设计、投票、排行、领奖核销全流程。

## 二、核心功能需求

### 2.1 用户认证与区域校验
- 手机号 + 验证码登录
- 用户位置/手机号归属地自动匹配赛区（10km 范围限定）
- 不在服务范围用户提示"暂未开放"

### 2.2 首页
- 活动主题 Banner + 当前赛区倒计时
- "立即生成我的蛋糕"核心 CTA
- Top100 免费领蛋糕说明
- 热门作品轮播流
- 规则摘要折叠区
- 底部导航：首页 / 排行榜 / 我的

### 2.3 AI 蛋糕生成
- 选择场景（生日/儿童/节庆/婚庆/其他）
- 输入主题关键词、祝福语、偏好色系、风格偏好
- 点击生成，展示 5 款设计效果图
- 支持重新生成
- 选择一个方案后进入发布确认

### 2.4 作品发布与分享
- 发布参赛作品（标题自动生成，可微调）
- 生成分享海报/分享卡片
- 显示作品编号与初始票数
- 分享到微信/朋友圈/社群

### 2.5 作品详情与投票
- 作品大图展示 + 当前票数 + 排名
- 投票按钮（1人1天3票限制）
- 投票受限时提示剩余票数与原因
- 分享拉票

### 2.6 排行榜
- Top 榜单列表（有效票排序）
- 我的排名浮层
- 赛区筛选
- 有效票说明

### 2.7 领奖核销
- 获奖后生成领奖码（二维码）
- 领奖门店地址 + 距离
- 领取截止时间
- 核销状态追踪

### 2.8 我的页面
- 我的作品列表
- 我的投票记录
- 我的领奖码与核销状态
- 客服与申诉入口

## 三、验收标准 (AC)

### AC-01: 用户登录与赛区校验
- 用户可通过手机号验证码登录
- 系统自动根据手机号归属地/定位匹配赛区
- 不在 10km 范围内提示"暂未开放"并展示可参与赛区列表

### AC-02: AI 蛋糕生成
- 用户提交需求后 30 秒内展示 5 张生成效果图
- 生成失败时展示友好错误提示
- 每人每小时限生成 5 次

### AC-03: 作品发布与分享
- 用户选择效果图后可在 3 步内完成发布
- 发布成功自动生成分享海报
- 作品立即出现在排行榜

### AC-04: 投票交互
- 用户可为其他作品投票，每日限 3 票
- 投票成功实时更新排行榜数据
- 风控拦截时提示冻结原因

### AC-05: 排行榜展示
- 排行榜展示有效票排序的 Top 作品
- 缓存延迟控制在 3-10 秒
- 支持赛区筛选

### AC-06: 领奖核销
- Top100 获奖用户收到通知并生成领奖码
- 领奖码为二维码形式
- 核销状态实时可查

### AC-07: iOS & Android 双端一致性
- iOS 和 Android 端功能完全一致
- UI 适配各平台设计规范（iOS Human Interface / Material Design）
- 核心交互不超过 3-5 步

## 四、技术约束
- 跨端方案：React Native（一套代码双端运行）
- 后端对接：复用现有 Rust Axum API
- 图片加载：支持 OSS 图片 CDN
- 离线能力：领奖码支持离线短时缓存
- 网络适配：村镇弱网环境下核心操作可用

## 五、非功能需求
- 冷启动时间 ≤ 3 秒
- 页面切换流畅度 ≥ 60fps
- 弱网（2G/3G）核心路径可用
- 内存占用 ≤ 150MB
- 支持 iOS 14+ 和 Android 8+
