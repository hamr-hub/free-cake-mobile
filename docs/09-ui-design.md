# 界面方案

## 一、设计目标
围绕“村镇用户低门槛参与 + 云端中台高效率统管”设计两套界面：
- C 端：简单、直给、强引导、低学习成本
- B 端：高信息密度、可配置、可审计、可调度

## 二、设计原则
- 关键路径最短：参赛、投票、核销都不超过 3-5 步
- 信息前置：明确赛区、规则、奖品、截止时间
- 风险可感知：异常票、异常核销、低库存需要显著提示
- 移动优先：C 端以小程序竖屏体验为主
- 运营优先：B 端后台支持批量操作和模板复用

## 三、C 端界面设计

### 1. 首页
目标：让用户快速理解活动并进入参赛。

模块：
- 顶部 Banner：活动主题、当前赛区、倒计时
- 核心 CTA：立即生成我的蛋糕
- 获奖说明：Top100 免费领 6 寸蛋糕
- 规则摘要：参与范围、投票规则、领取规则
- 热门作品轮播
- 我的状态入口：我的作品 / 我的票数 / 我的领奖码

页面结构：
1. 头图活动区
2. 当前赛区与定位确认
3. 主按钮
4. 热门作品流
5. 规则说明折叠区
6. 底部导航

### 2. AI 生成页
目标：帮助用户快速生成可参赛设计。

表单项：
- 场景：生日 / 儿童 / 节庆 / 婚庆 / 其他
- 主题关键词
- 祝福语
- 偏好色系
- 风格偏好

交互：
- 输入后点击生成
- 展示“正在生成 5 款设计”进度
- 生成结果宫格展示
- 支持重生成
- 选择一个方案后进入发布确认

### 3. 作品发布页
模块：
- 已选蛋糕图
- 标题自动生成，可微调
- 参赛声明确认
- 发布按钮

说明：
- 发布后生成分享海报 / 分享卡片
- 展示作品编号与当前初始票数

### 4. 作品详情页
模块：
- 大图展示
- 当前票数
- 排名
- 分享按钮
- 投票按钮
- 风险提示文案
- 活动倒计时

说明：
- 若用户是作者，展示“查看我的领奖状态”
- 若投票受限，直接提示剩余票数与原因

### 5. 排行榜页
模块：
- Top 榜单
- 我的排名浮层
- 赛区筛选
- 有效票说明
- 申诉入口

### 6. 我的页面
模块：
- 我的作品
- 我的投票记录
- 我的领奖码
- 核销状态
- 客服与申诉入口

### 7. 领奖核销页
模块：
- 领奖码二维码
- 领奖门店
- 领取截止时间
- 手机号校验说明
- 领取须知

## 四、B 端后台界面设计

### 1. 总览 Dashboard
目标：总部快速看到全域经营状态。

核心卡片：
- 活动中赛区数
- 今日新增参赛数
- 今日投票数
- 风险票占比
- 待排产订单数
- 今日核销率
- 低库存门店数

图表：
- 各赛区参与趋势
- 投票趋势
- 核销漏斗
- 门店产能利用率

### 2. 赛区管理页
字段：
- 赛区名称
- 覆盖范围
- 绑定门店
- 活动状态
- 当前人数
- 风险等级

操作：
- 新建赛区
- 配置服务半径
- 绑定门店
- 上下线

### 3. 活动管理页
模块：
- 活动列表
- 活动模板
- 时间配置
- 奖品规则
- 投票规则
- 发布 / 下线 / 复制活动

### 4. 作品审核页
模块：
- 待审核作品列表
- AI 图预览
- 风险标签
- 一键通过 / 驳回
- 批量处理

### 5. 投票风控台
目标：集中处理刷票与异常。

模块：
- 异常事件列表
- 风险分布图
- 设备 / IP 聚类视图
- 票数冻结记录
- 扣票操作台
- 审计日志

关键交互：
- 点击作品查看投票明细
- 一键冻结 / 解冻
- 批量扣减异常票

### 6. 开奖与订单页
模块：
- 活动结算结果
- Top100 名单
- 获奖资格复核
- 奖励订单生成状态
- 异常订单重试

### 7. 排产中心
模块：
- 门店产能看板
- 待生产订单池
- 生产批次列表
- 拖拽排产
- 生产异常标记

### 8. 核销管理页
模块：
- 当日待领取
- 已领取
- 失败核销记录
- 重发核销码
- 超时未领处理

### 9. 库存中心
模块：
- 门店库存列表
- 安全库存预警
- 原料台账
- 出入库记录
- 报损单

### 10. 门店管理页
模块：
- 门店信息
- 设备状态
- 营业状态
- 联系人
- 最近异常

### 11. 人员与考勤页
模块：
- 员工列表
- 班次
- 打卡记录
- 培训完成度
- 薪资核算状态

## 五、门店端界面设计

### 1. 门店工作台首页
- 今日待生产数量
- 今日待核销数量
- 设备状态
- 低库存提醒
- 异常任务提醒

### 2. 生产任务页
- 任务列表
- 优先级
- 计划时间
- 模板图预览
- 开始 / 暂停 / 完成 / 异常上报

### 3. 核销页
- 扫码输入区
- 手机号后二次确认
- 核销结果提示
- 异常处理入口

### 4. 库存盘点页
- 原料列表
- 实盘数量
- 差异上报
- 补货申请

## 六、页面流转
### C 端主路径
首页 → AI 生成 → 选图发布 → 作品详情 → 分享拉票 → 开奖通知 → 领奖码 → 到店核销

### B 端主路径
Dashboard → 活动配置 → 监控参赛与投票 → 风控处置 → 开奖 → 生成订单 → 排产 → 监控核销

### 门店端主路径
工作台 → 查看任务 → 执行生产 → 完工确认 → 扫码核销 → 异常上报

## 七、关键组件建议
- 倒计时横幅
- 作品卡片
- 排名徽标
- 风险标签
- 审计抽屉
- 核销状态弹窗
- 产能日历
- 库存预警标签

## 八、视觉风格建议
### C 端
- 风格：轻快、暖色、节庆感
- 主色：奶油黄 / 草莓粉 / 焦糖棕
- 强调“免费领”“Top100”“到店自提”

### B 端
- 风格：简洁、数据驱动、低干扰
- 主色：深蓝 / 中性灰
- 风险类信息统一橙 / 红警示

## 九、低保真原型建议
建议下一步输出：
- C 端：首页、AI 生成页、详情页、领奖页
- B 端：Dashboard、活动管理页、风控台、排产中心
- 门店端：工作台、生产任务页、核销页
